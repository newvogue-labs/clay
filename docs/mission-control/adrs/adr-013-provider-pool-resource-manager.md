# ADR-013: Provider-Pool как resource-manager (homo/hetero модель)

- **Status:** Proposed
- **Date:** 2026-06-13
- **Связь:** расширяет ADR-005 (model-provider-abstraction) и ADR-009 (egress-gateway);
  blueprint §10; готовит код-слайс stateful provider-pool; мотивы conduit-паттернов.

## Контекст

Free-провайдеры/ключи нестабильны: 429 (RPD/RPM), geo-400, transient. Назначение модели роли
«один клиент на ctor» хрупко при отвале ключа.

## Решение

Stateful provider-pool в схеме `ops`: резервуар ключей + health-tracking.
- Гетерогенность МЕЖДУ ярусами; гомогенный ключ-пул ВНУТРИ яруса (ротация + health).
- Fallback-политика через LiteLLM `fallbacks`.
- Процедура быстрой замены отвалившейся модели/ключа без смены контракта роли.

## Последствия

- (+) Устойчивость к падению ключа/модели; предсказуемый RPD-бюджет на ярус.
- (+) Готовит резервуар под несколько ролей одного яруса.
- (−) Новое stateful-состояние в `ops` → миграция + retention (отдельный код-слайс).

## Политика

Демо = любые free через шлюз; real-money = только официальные платные.
