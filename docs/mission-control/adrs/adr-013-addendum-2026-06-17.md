# ADR-013 — Addendum (2026-06-17): граница интеграции и схема stateful provider-pool

**Status:** Accepted (addendum к Proposed ADR-013; ратифицирует интеграцию по итогам recon S0).

**Context (recon S0, 2026-06-17):**

- Выбор модели: `role_id → ServiceModelResolver.resolve_model_id → assignments[role_id] → RoutingModelClient → LiteLLM :4000`. `model_name ≡ model_id` 1:1.
- `config.yaml` пишется руками (нет шаблонизации/`yaml.dump`/jinja).
- LiteLLM: `usage-based-routing-v2`, cooldown/usage **in-memory** (не переживает рестарт), не знает дневных окон (RPD/daily-token) и не моделирует ключи как ресурсы; Redis не используется.
- Найден референс `~/Projects/conduit/gateway/account_pool.py` (машина состояний ключа).

**Decision:**

1. **Вариант A (reconcile-config).** Пул — источник правды; периодический reconcile-loop генерирует `model_list` из **доступных** деплоев и пишет `config.yaml` + hot-reload (SIGHUP). Горячий путь Clay не меняется. Per-request fallback остаётся за LiteLLM. `assignments` неизменны.

2. **Машина состояний ключа** (адаптация Conduit `AccountPool`):
   - `available → cooling`: rate-limit/`blocked`/мягкие отказы ≥ `allowed_fails` (cooldown, дефолт 6h / `21600s`).
   - `available|cooling → exhausted`: исчерпана дневная квота (RPD или daily-token). Восстановление — по `reset_at` провайдера (НЕ фикс. 24h: напр. LongCat 19:00 МСК / Asia-Shanghai, Google per-account).
   - `* → dead`: `auth_fail` (401/403). Авто-восстановления НЕТ (manual revive + алерт Emma).
   - `cooling → available`: по истечении `cooling_until`. `exhausted → available`: по `reset_at`.
   - outcome-классы: `success / auth_fail / quota / blocked / timeout / bad_request / upstream`.

3. **Секреты в БД НЕ хранятся.** `provider_key.key_ref` = имя env-переменной (напр. `GOOGLE_AISTUDIO_KEY_3`); реальные ключи остаются в `litellm.env` (600). Reconcile подставляет `os.environ/${VAR}` в `config.yaml`.

4. **Гео/trust (ADR-009)** — атрибут провайдера (`realmoney_ok | demo_only`); reconcile в demo-режиме включает все, в real-money — только `realmoney_ok`.

**Schema (ops):** `provider`, `provider_key` (состояние+квоты), `provider_deployment` (model_name↔key↔upstream-slug+params), `provider_health` (hypertable: outcome/latency/tokens, audit + питание машины состояний).

**Consequences:** 0 кода в горячем пути; задержка реакции = период reconcile; live 5432 не трогаем (миграции на 5433). Decision-замена прежней пометки «gateway-native = промежуток».
