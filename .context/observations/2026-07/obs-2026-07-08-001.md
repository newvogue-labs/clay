name: Notion API DPI-блокировка на хосте
description: api.notion.com молчаливо дропает TLS handshake с хозяина clay по IPv4 и IPv6
type: block
---

**Контекст:** S2-APPLY-1 — полный `--apply` vault→Notion (57 карт). Предыдущий dry-run (сеть не трогает) успешен: 57 CREATE. `--apply` упал с ConnectTimeout (SSL handshake) на `find_page_by_clay_id`.

**Диагностика:**
- `curl -4 --max-time 10 https://api.notion.com/v1/users/me` → timeout (exit 28), client hello уходит, ответа нет
- `curl -6` → exit 7 (IPv6 сеть недоступна)
- `CLAY_NOTION_FORCE_IPV4=1` (httpx `local_address="0.0.0.0"`) не помогает — IPv4 тоже режется
- Поведение соответствует DPI-блокировке по SNI (api.notion.com)

**Решение:** Не строить DPI-обход (v2ray/clash/WireGuard) на боевом торговом сервере ради advisory-зеркала. Полный `--apply` прогнать с ноута Emma.

**Why / How to apply:**
- При следующем S2-APPLY-1 запускать с машины, где `curl -4` даёт 401
- Не пытаться чинить сеть на хосте
- Для CI/CD-автоматизации — вынести в GitHub runner с доступом к Notion
