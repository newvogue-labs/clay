## Правило

Когда в одном сообщении от архитектора есть **два потока**:
1. Задачи для агента (код, команды, диффы)
2. Действия, которые должна сделать Emma сама (прислать файл, принять решение, запустить команду)

**Агент обязан:**
- Выполнять свою часть (tool calls)
- **Явно выделять "Твоя часть"** в ответе — конкретный список: что прислать, какой файл, какое решение принять
- **НЕ запускать tool calls за Emma** (не читать файлы за неё, не запускать bash за неё)

**Формат ответа:**
```
[Моя часть — выполняю]
- Здесь что я сделал(а) (команды, код, применение diffs)

---
Твоя часть:
- Конкретный список: что прислать, какой файл, какое решение принять
- Без tool calls с моей стороны
```

**Пример:**
```
[Моя часть]
Применяю дифф 1c-a-1 → тестирую → коммичу

---
Твоя часть:
Пришли полный файл `binance_testnet.py` (HEAD 47ad378)
и подтверди, существует ли `tests/execution/test_binance_testnet.py`
```

**Правило:**
Если в исходном сообщении есть "пришли файл" / "подтверди" / "реши" / "проверь" — это **ТВОЯ часть**. Агент ждёт, а не запускает инструменты за неё.
